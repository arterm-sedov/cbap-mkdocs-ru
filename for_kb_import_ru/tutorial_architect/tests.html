Урок 1. Введение, рабочий стол, роли, навигация, дашборды

Практические задания

1.1. Роли и права доступа

  • Определите, какие возможности должны быть доступны роли «Архитектор» в модуле корпоративной архитектуры. Запишите свой список и объясните, почему выбрали именно эти действия.

1.2. Элементы рабочего стола

  • Найдите на рабочем столе элемент, который предоставляет быстрый доступ к созданию новых объектов архитектуры. Опишите, как бы вы использовали его в своей работе.

1.3. Работа с отчётами

  • Расположите действия в правильном порядке для анализа данных в отчётах: применение фильтров, переход к детализации, открытие отчёта, экспорт результатов.

1.4. Самостоятельное задание

  • Опишите последовательность действий для получения отчёта по оргединицам с фильтрацией по конкретному подразделению. Какие шаги вы бы предприняли?

Урок 2. Формирование и визуализация ОШС, справочники, сотрудники, документы

Практические задания

2.1. Справочники

  • Для чего используется справочник юридических лиц в контексте организационной структуры? Приведите пример.

2.2. Импорт оргединиц

  • На какие аспекты необходимо обратить особое внимание при импорте оргединиц из Excel? Составьте свой чек-лист.

2.3. Автоматизация форм

  • Что НЕ может вычисляться автоматически при работе с оргединицами? Объясните свой выбор.

2.4. Самостоятельное задание

  • Создайте справочник «Типы подразделений» с атрибутами: название, код, описание. Опишите пошаговый алгоритм.

Урок 3. Моделирование процессной архитектуры, интеграция, публикация

Практические задания

3.1. Иерархия процессов

  • Расположите элементы процессной архитектуры в правильной иерархии (от верхнего уровня к нижнему): BPMN-диаграммы, диаграммы бизнес-способностей, группы процессов.

3.2. BPMN-элементы

  • Какие элементы НЕ относятся к базовой палитре редактора диаграмм? Объясните, почему.

3.3. Связи в процессах

  • Что можно связать с задачами в BPMN-диаграмме? Приведите примеры.

3.4. Самостоятельное задание

  • Опишите процесс импорта BPMN-диаграммы и настройки связей с оргструктурой. Какие шаги вы бы предприняли?

Урок 4. Согласование, аудит, публикация, контроль изменений

Практические задания

4.1. Процесс согласования

  • В каком порядке выполняются этапы согласования модели? Опишите свой вариант.

4.2. Автоматизация назначения задач

  • По каким принципам происходит автоматическое назначение задач согласования? Приведите примеры.

4.3. Аудит процессов

  • Что НЕ входит в задачи аудитора при выполнении аудита процесса? Объясните свой выбор.

4.4. Самостоятельное задание

  • Опишите, как использовать аналитические отчёты для анализа эффективности процессов согласования. Какие метрики вы бы использовали?

Урок 5. Документооборот, шаблоны, экспорт, интеграция, лучшие практики

Практические задания

5.1. Шаблоны экспорта

  • Какие форматы поддерживаются для шаблонов экспорта документов? Приведите примеры использования.

5.2. Структура шаблона

  • Что обязательно должно быть указано в шаблоне для корректной генерации документов? Почему это важно?

5.3. Интеграция данных

  • Какие форматы поддерживаются для экспорта данных в целях интеграции с внешними системами? Какой бы вы выбрали и почему?

5.4. Лучшие практики

  • Какие действия НЕ относятся к лучшим практикам управления документооборотом? Объясните свой выбор.

5.5. Самостоятельное задание

  • Разработайте план внедрения автоматизированного документооборота для процесса «Адаптация нового сотрудника». Какие шаги и метрики вы бы предложили?

Итоговое практическое задание

Сценарий: ваша организация планирует внедрить модуль корпоративной архитектуры. Вам необходимо:

  1. Создать организационную структуру с 3 уровнями иерархии
  2. Импортировать данные о сотрудниках
  3. Создать диаграмму бизнес-способностей «Управление персоналом»
  4. Настроить процесс согласования изменений
  5. Подготовить шаблоны для экспорта должностных инструкций

Вопросы:

  1. Опишите последовательность действий для выполнения задач.
  2. Какие справочники необходимо создать в первую очередь?
  3. На что обратить внимание при подготовке Excel-файла для импорта?
  4. Какие элементы должны быть включены в диаграмму бизнес-способностей «Управление персоналом»?
  5. Кого назначить согласующими для изменений в HR-процессах?
  6. Какие системные поля включить в шаблон должностной инструкции?

  3. Внимание: сопоставление столбцов, корректность данных, структура дерева

  5. Согласующие: руководители, HR, заинтересованные лица

Введение

Здесь представлены инструкции по обновлению версии экземпляра ПО Comindware Platform с 4.7.2 до версий 4.7.3–5.0 для ОС Linux.

Порядок обновления версии экземпляра ПО

Обновить версию экземпляра ПО можно одним из двух способов:

  • обновить контейнеры с ПО;
  • обновить Helm-чарт Comindware Platform.

Обновление контейнеров

  1. Сохраните резервную копию экземпляра ПО. См. «Резервное копирование. Настройка и запуск, просмотр журнала сеансов».

  2. Выполните команду:

    helm upgrade --set image.name=<SET-NAME> --set image.tag=<SET-VERSION> <HELM-RELEASE-NAME> <HELM-CHART>

    Здесь:

    • <SET-NAME> — имя контейнера для обновления;
    • <SET-VERSION> — целевая версия контейнера;
    • <HELM-RELEASE-NAME> — имя Helm-релиза для Comindware Platform;
    • <HELM-CHART> — путь к директории (./comindware-v.5.W.XYZ.0/), путь к файлу .tgz чарта или FQDN-имя Helm-чарта.

  3. Убедитесь, что обновление было выполнено успешно:

    helm list 
    kubectl get pods

  4. В браузере откройте URI Ingress-эндпоинта с Comindware Platform.

  5. Если отобразится страница режима обслуживания, инициализируйте экземпляр ПО. См. «Инициализация Comindware Platform».

Обновление Helm-чарта

  1. Сохраните резервную копию экземпляра ПО.

  2. Ознакомьтесь с разницей между текущей и новой версией чарта:

    helm diff upgrade <HELM-RELEASE-NAME> <TARGET-HELM-CHART>

    Здесь:

    • <HELM-RELEASE-NAME> — имя Helm-релиза для Comindware Platform;
    • <TARGET-HELM-CHART> — путь к директории (./comindware-v.5.W.XYZ.0/), путь к файлу .tgz чарта или FQDN-имя Helm-чарта.

  3. Выполните обновление с помощью команды:

    helm upgrade <HELM-RELEASE-NAME> <TARGET-HELM-CHART>

  4. Убедитесь, что обновление было выполнено успешно:

    helm list 
    kubectl get pods

  5. В браузере откройте URI Ingress-эндпоинта с Comindware Platform.

  6. Если отобразится страница режима обслуживания, инициализируйте экземпляр ПО. См. «Инициализация Comindware Platform».

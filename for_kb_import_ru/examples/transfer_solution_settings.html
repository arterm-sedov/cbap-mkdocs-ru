Экспериментальная функция

Представленная здесь функция находится на стадии разработки. См. «Поддержка экспериментальных функций».

Здесь приведены основные способы переноса настроек приложения между стендами Comindware Platform.

Обычно для разработки, тестирования и промышленной эксплуатации используют разные стенды. Чтобы поддерживать их в согласованном состоянии, настройки приложения переносят из среды разработки в тестовую, а затем в промышленную среду.

Способы переноса

Выберите способ переноса в зависимости от задачи: требуется ли перенести только конфигурацию приложения или весь стенд вместе с данными.

Способ Когда использовать Что переносится
Перенос базы данных Требуется перенести стенд целиком или восстановить его состояние из резервной копии. Все настройки стенда и все данные.
Ручная настройка Приложение небольшое, а перенос требуется выполнить один раз без подготовки версии приложения. Только те настройки и данные, которые администратор воссоздаёт вручную.
Экспорт и импорт версии приложения Требуется регулярно переносить изменения между стендами и контролировать версии приложения. Конфигурация приложения. Записи переносятся только для шаблонов, настроенных на перенос данных при трансфере.

Перенос базы данных

При переносе базы данных переносятся все настройки стенда и все данные. Этот способ подходит для полного восстановления стенда или создания копии среды.

См. «Резервное копирование и восстановление. Содержание раздела».

Ручная настройка

Ручная настройка означает, что администратор заново создаёт приложение, шаблоны, атрибуты, формы, роли и другие элементы на целевом стенде.

Этот способ подходит только для небольших приложений или разовых работ, так как он требует больше времени и повышает риск расхождений между стендами.

Экспорт и импорт версии приложения

Для регулярного переноса настроек используйте управление версиями приложения:

  • ручной экспорт и импорт файла CTF;
  • экспорт и импорт через Git;
  • экспорт и импорт через Excel.

Этот способ быстрее ручной настройки и позволяет контролировать изменения между версиями приложения.

Примечание

По умолчанию при экспорте версии приложения переносятся настройки приложения. Записи переносятся только для шаблонов, у которых включён перенос данных при трансфере.

См. «Управление версиями приложения. Экспорт и импорт».

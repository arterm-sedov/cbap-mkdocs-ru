Внимание!

Если ваша версия Comindware Platform выпущена после 15.01.2026, её поведение может отличаться от версии, выпущенной 31.12.2025.

Введение

Comindware Platform версии 5.0 от 31.12.2025 включает ряд новых возможностей, улучшений и исправлений по сравнению с версией 5.0 от 10.10.2025.

Ключевые изменения

  • Настройка создания топиков Apache Kafka: добавлены параметры конфигурации для управления созданием топиков очереди сообщений.

    См.: Конфигурация экземпляра, компонентов ПО и служб. Настройка

  • Поддержка сервисов на Service Host (API gateway): реализована поддержка частичного и полного переопределения асинхронных сервисов, поддержка Hub.

  • Увеличение времени ожидания ответа адаптеров: увеличено время ожидания ответа для HTTP-адаптера до 10 минут и для MySQL-адаптера до 5 минут.

  • Механизм автоматического запуска адаптеров: реализован механизм автоматического запуска адаптеров на старте AdapterHost, включая адаптеры, которые платформа уже отправляла.

  • Обсуждения: улучшена система обсуждений: добавлены системные уведомления о новых сообщениях, обновление счётчиков непрочитанных сообщений, перенос сервиса подписок в API gateway.

Новые возможности и улучшения

Интерфейс

  • Атрибут типа «Цвет»: добавлен виджет выбора цвета для мобильного приложения.

  • Рабочий стол и дашборды:

    • исправлено отображение высоты графиков на странице рабочего стола;
    • удалён лишний скролл в настройке страницы боковой навигации;
    • исправлено сохранение быстрых фильтров в дашбордах;
    • улучшена валидация быстрых фильтров;
    • исправлено отображение атрибутов в настройке детализации данных диаграммы;
    • улучшена видимость цвета прошедших токенов в процессах.

  • Фильтры: оптимизирована работа редактора выпадающих списков для уменьшения нагрузки на систему при работе с фильтрами ссылочных атрибутов.

  • Навигация: исправлено дублирование виджетов в колонках при редактировании страниц навигации.

  • Таблицы и коллекции: реализован перенос слов в заголовках атрибутов коллекций для улучшения отображения длинных названий.

  • Шаблоны экспорта: исправлена выгрузка таблиц с одним столбцом в Word.

Инфраструктура

  • Очередь сообщений: добавлены параметры конфигурации для управления созданием топиков Apache Kafka:

    • mq.replicationFactor — параметр ReplicationFactor для создаваемого топика;
    • mq.numPartitions — параметр NumPartitions для создаваемого топика;
    • mq.metadataTimeoutMsec — таймаут для запроса метаданных.

  • База данных:

    • реализован сбор объектов через Object API с использованием нативных запросов;
    • реализован набор данных на нативных запросах;
    • улучшена конфигурация кэшей: добавлена возможность автоматического применения новой конфигурации к существующим кэшам при запуске системы.

  • Резервное копирование: добавлена валидация временных интервалов сессий резервного копирования для корректного выполнения по расписанию.

  • Системные уведомления: уменьшен размер сообщений по WebSocket путём удаления null-значений из сериализатора процессора запросов.

  • Кластер: реализована остановка платформы при переходе кластера в неактивное состояние или режим только чтения.

  • Сессии: реализована поддержка идентификатора устройства в сессиях.

Мобильное приложение

  • Атрибуты:

    • исправлено сохранение значений HTML-атрибутов в мобильном приложении;
    • исправлено отображение значений в карточках: значения теперь корректно уходят в троеточие при переполнении;
    • исправлено отображение троеточия с функциями при наличии одной картинки в атрибуте типа «Изображение»;
    • добавлен виджет выбора цвета для атрибута типа «Цвет».

  • Кнопки и операции:

    • исправлена работа кнопок в коллекциях;
    • исправлено отображение кнопок, удовлетворяющих условию отображения;
    • исправлена ошибка создания связанной записи по кнопке при наличии диалогового окна;
    • исправлено разное отображение кнопок в мобильном приложении и десктопе для форм только для чтения.

  • Графики и диаграммы: исправлено падение мобильного приложения при наличии столбчатого графика на рабочем столе.

  • Интерфейс: исправлены стили на странице загрузки.

Исправления

  • Регламенты и процессы:

    • исправлено обновление ссылок при переводе процесса в исполняемый: теперь корректно отображаются PNG-изображения по вложенному регламенту;
    • исправлено хранение значений атрибутов моделей: атрибуты моделей больше не хранят значения всех версий диаграмм;
    • исправлена потеря очереди токенов (зависания) без перезапуска.

  • Формы:

    • исправлено сохранение правил форм и фильтров виджетов в формах пользовательских задач с системными атрибутами шаблона процесса;
    • исправлено сохранение правил на формах пользовательских задач.

  • Обновление и установка:

    • исправлено обновление с версии 4.7.23499.0 на 5.0.24055.0;
    • исправлено обновление для аккаунтов с одинаковыми именами пользователей с разными регистрами (case-insensitive).

  • Лицензирование:

    • исправлен некорректный подсчёт статистики GetLicenseStatus;
    • исправлен статус isLicensed: true при авторизации без лицензий.

  • Аутентификация: исправлена неверная иконка указателя мыши на ссылках страницы входа.

  • Производительность и стабильность:

    • исправлена обработка необработанных исключений на таймерах;
    • исправлены опечатки в логах;
    • исправлены скрипты отчётов;
    • оптимизирована инициализация сервиса адаптеров и сокращено время выполнения тестов кластера.

# HTTP-запросы типа GET. Отправка запроса и обработка ответа в формате JSON. Настройка подключения, пути передачи данных и сценария

## Введение

**Comindware Platform** позволяет отправлять HTTP-запросы типов `GET`, `POST`, `PUT`, `DELETE` для взаимодействия с внешними системами.

Здесь представлен пример настройки подключения, пути передачи данных и [сценария](https://kb.comindware.ru/article.php?id=5654) для отправки HTTP-запросов типа `GET` и записи данных из ответа сервера в формате JSON в атрибуты шаблона записи.

Настройка для обработки данных в формате XML и простого текста будет аналогичной.

## Прикладная задача

- Имеется приложение для работы с клиентами и заказами.
- В приложении настроены шаблоны:
  - *«Клиенты»* — для хранения данных клиентов.
  - *«Заказы»* — для хранения сведений о заказах, полученных из внешней системы.
- Имеется внешняя система управления заказами, предоставляющая данные заказов по HTTP-запросу.
- Требуется настроить приложение так, чтобы пользователь мог открыть карточку клиента и нажать кнопку *«Получить заказы»*.
- По нажатию кнопки **Comindware Platform** должен отправлять GET-запрос во внешнюю систему, получать список заказов выбранного клиента, создавать для каждого заказа запись в шаблоне *«Заказы»* и связывать её с текущей записью клиента.
- Путь для запроса данных заказов: `http://<your-hostname>/api/orders`

  Примечание

  В данной статье рассматривается настройка отправки GET-запросов на URL `http://<your-hostname>/api`.

  Во всех примерах подставьте реальный адрес своего сервера вместо `<your-hostname>`.
- Запрос содержит параметр `clientName` с идентификатором клиента.
- В ответ на запрос сервер возвращает JSON-ответ с массивом данных заказов следующего вида:

```
{
    "error": false,
    "status": "ok",
    "statusCode": 200,
    "orders": [
        {
            "orderID": 1,
            "orderStatus": "Fulfilled",
            "orderAmount": 1000
        },
        {
            "orderID": 2,
            "orderStatus": "Canceled",
            "orderAmount": 5000
        },
        {
            "orderID": 3,
            "orderStatus": "New",
            "orderAmount": 3500
        }
    ]
}
```

## Порядок настройки

1. Настройте [подключение](#http_send_example_connection) типа «**Отправка HTTP-запросов**».
2. Настройте [путь передачи данных](#http_send_example_communication_route) типа «**Отправка HTTP-запросов**».
3. Настройте [шаблон](#http_send_example_record_template) для записи полученных данных.
4. Настройте [сценарий](#http_send_example_scenario), который будет формировать и отправлять HTTP-запрос на сервер, создавать записи заказов по данным из ответа и связывать их с текущим клиентом с использованием настроенных подключения и пути передачи данных.

## Настройка подключения

1. Откройте раздел «**Администрирование**» — «**Подключения**».
2. В меню «**Создать**» выберите пункт «**Подключения REST и OData**» — «**Отправка HTTP-запросов**».

   ![Создание подключения для отправки HTTP-запросов](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_create.png)

   Создание подключения для отправки HTTP-запросов
3. Настройте подключение для отправки HTTP-запросов со следующими параметрами:

   - **URI** — `http://<your-hostname>/api`. Символ `/` в конце URI не требуется.
   - **Формат данных** — **JSON**. Также поддерживаются **XML** и **простой текст**, но здесь рассматривается обработка данных в формате JSON.
4. Остальные параметры подключения настройте в соответствии с используемым сервером. См. статью «*[Отправка HTTP-запросов. Настройка подключения](https://kb.comindware.ru/article.php?id=5633)*».

_![Настройка подключения для отправки HTTP-запросов](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_settings.png)_

## Настройка пути передачи данных

1. Откройте раздел «**Администрирование**» — «**Пути передачи данных**».
2. В меню «**Создать**» выберите пункт «**Подключения REST и OData**» — «**Отправка HTTP-запросов**».

   ![Создание пути передачи данных для отправки HTTP-запросов](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_path_create.png)

   Создание пути передачи данных для отправки HTTP-запросов
3. Настройте свойства пути передачи данных на следующих вкладках:

   - [**Основные свойства**](#http_send_example_basic_properties)
   - [**Атрибуты сообщения**](#http_send_example_message_attributes)
   - [**Интеграция**](#http_send_example_integration)
4. Сохраните путь передачи данных.

### Основные свойства

- **Подключение** — выберите ранее созданное [подключение для отправки HTTP-запросов](#http_send_example_connection).
- **Системное имя** — введите системное имя пути передачи данных латинскими буквами.
- **Отключить** — установите этот флажок, если требуется деактивировать путь передачи данных.
- **Описание** — введите наглядное описание назначения пути передачи данных.

_![Настройка основных свойств пути передачи данных для отправки HTTP-запросов](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_route_general_settings.png)_

### Атрибуты сообщений

- **Тип сообщения** — выберите пункт «**Отправка HTTP-запросов**».
- **Запрос** — добавьте атрибут `queryParams` (произвольное системное имя, которое будет указано в поле «**Атрибут для параметров запроса**» на вкладке «[**Интеграция**](#http_send_example_integration)» и в действии сценария «**Изменить значения переменных**») типа «**Объект**».

  - Установите флажок «**Массив**».
  - Установите флажок у атрибута `queryParams` и добавьте его дочерние атрибуты:

    - `name` (предопределённое системное имя) типа «**Строка**» — этот атрибут будет содержать имена параметров запроса.
    - `value` (предопределённое системное имя) типа «**Строка**» — этот атрибут будет содержать значения параметров запроса.
    - Установите флажок «**Массив**» у атрибута `value`.

    В эти атрибуты будут передаваться пары ключ-значение параметров запроса.
- **Ответ** — добавьте атрибут с системным именем `orders` (совпадает с именем массива данных заказов в ответе сервера на HTTP-запрос) типа «**Объект**».

  - Установите флажок «**Массив**»
  - Установите флажок у атрибута `orders` и добавьте его дочерние атрибуты:
    - `orderAmount` (совпадает с именем параметра из ответа сервера) типа «**Число**» — этот атрибут будет содержать сумму заказа.
    - `orderID`  (совпадает с именем параметра из ответа сервера) типа «**Число**» — этот атрибут будет содержать ID заказа.
    - `orderStatus` (совпадает с именем параметра из ответа сервера) типа «**Строка**» — этот атрибут будет содержать статус заказа.
      В эти атрибуты будут записываться значения из ответа сервера на запрос.
- **Ответ с ошибкой** — настройте атрибуты для получения ответа сервера с ошибкой аналогично атрибутам **запроса** и **ответа**. Например, `errorResponse` типа «**Строка**».

_![Настройка атрибутов сообщения в свойствах пути передачи данных для отправки HTTP-запросов](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_message_attributes_settings.png)_

### Интеграция

- **Метод запроса** — выберите пункт «**GET**».
- **Шаблон пути запроса** — введите `orders` (фактический путь к HTTP-запросу на сервере), этот суффикс будет добавлен к **URI**, указанному в [свойствах подключения для отправки HTTP-запросов](https://kb.comindware.ru/article.php?id=5633). Например, для **URI** `http://<your-hostname>/api` результирующий путь запроса будет `http://<your-hostname>/api/orders`.
- **Атрибут для сериализации в тело запроса** — оставьте пустым, так как в данном примере используется метод `GET` без тела запроса.
- **Атрибут для параметров запроса** — введите системное имя атрибута из таблицы «**Запрос**» на вкладке «**Атрибуты сообщений**». В данном случае, введите `queryParams`.

  ![Настройка интеграции в свойствах пути передачи данных для отправки HTTP-запросов](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_integration_settings.png)

  Настройка интеграции в свойствах пути передачи данных для отправки HTTP-запросов
- **Атрибуты для десериализации ответа без ошибки** и  **Атрибуты для десериализации ответа с ошибкой**

  - Создайте одну строку в каждой таблице и введите символ `$` в столбцы «**Путь к атрибуту**» и «**Выражение на языке запросов**».

    ![Настройка атрибутов для десериализации ответа в пути передачи данных для отправки HTTP-запросов](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_attribute_serialization_settings.png)

    Настройка атрибутов для десериализации ответа в пути передачи данных для отправки HTTP-запросов

## Настройка шаблона записи

1. Создайте шаблон записи *«Заказы»* со следующими атрибутами:

   | Системное имя | Тип | Параметры |
   | --- | --- | --- |
   | *«ID\_заказа»* | **Число** | Установить флажок «**Использовать как заголовок записей**» |
   | *«Статус»* | **Текст** |  |
   | *«Сумма\_заказа»* | **Число** | Количество знаков после запятой: «**Не преобразовывать**» |
2. Создайте шаблон записи *«Клиенты»*.
3. Добавьте следующие атрибуты:

   | Системное имя | Тип | Параметры |
   | --- | --- | --- |
   | *«Название»* | **Текст** | Установить флажок «**Использовать как заголовок записей**» |
   | *«Заказы»* | **Запись** | («**Хранить несколько значений**») связанный с шаблоном «**Заказы**» с атрибутом *«Клиент»* |
4. Добавьте кнопку *«Получить заказы»* со следующими свойствами:

   - **Контекст операции:** запись
   - **Операция:** вызвать событие «Нажата кнопка»
   - **Результат выполнения:** обновить данные
5. Поместите атрибуты и кнопку на основную форму. Представление атрибута *«Заказы»* сделайте в виде таблицы и вынесите на неё атрибуты *«ID заказа»*, *«Статус»* и *«Сумма заказа»*.

## Настройка сценария отправки HTTP-запроса

1. Создайте [сценарий](https://kb.comindware.ru/article.php?id=5654), выполняющийся по нажатию кнопки. Выберите в настройках блока контекстный шаблон записи и кнопку (например, контекстный шаблон: `Клиенты`, кнопка: `Получить заказы`).

   ![Исходный сценарий, выполняющийся по нажатию кнопки](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_scenario_button_event_settings.png)

   Исходный сценарий, выполняющийся по нажатию кнопки
2. Добавьте и настройте действие «**Изменить значения переменных**».

   - **Операция со значениями переменных** — **Заменить**.
   - **Набор переменных** — `queryParamsContainer` (произвольное системное имя), из этого массива последующее действие «**Отправить сообщение**» передаст параметры в HTTP-запрос.
   - В таблице атрибутов создайте атрибут `queryParams` (это системное имя массива атрибутов **запроса**, указанное на вкладке «[**Атрибуты сообщений**](#http_send_example_message_attributes)» в свойствах пути передачи данных), который будет содержать пару ключ-значение для параметра HTTP-запроса в двух дочерних атрибутах:

     - `name` — введите формулу с именем параметра HTTP-запроса (например, `"clientName"`).
     - `value` — введите **формулу**, выражение на **N3** или выберите **атрибут** с необходимым значением параметра HTTP-запроса (например, атрибут `Имя`).

       Примечание

       Чтобы передать в HTTP-запрос несколько параметров, создайте необходимое количество атрибутов с одинаковым системным именем и настройте их дочерние атрибуты, например для массива из двух элементов:

       - `queryParams`
         - `name`
         - `value`
       - `queryParams`
         - `name`
         - `value`

       ![Настройка переменных для передачи в качестве параметров HTTP-запроса](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_scenario_query_params_settings.png)

       Настройка переменных для передачи в качестве параметров HTTP-запроса
3. Добавьте и настройте действие «**Отправить сообщение**».

   - **Подключение** — выберите [подключение для отправки HTTP-запросов](https://kb.comindware.ru/article.php?id=5633).
   - **Путь передачи данных** — выберите ранее настроенный [путь для передачи данных](#http_send_example_communication_route) HTTP-запроса.
   - **Переменная с сообщением** — введите системное имя **набора переменных**, указанное в предшествующем действии «**Изменить значения переменных**» (например, `queryParamsContainer`).
   - **Переменная для успешного ответа** — введите системное имя переменной, в которую будет помещён ответ сервера на HTTP-запрос (например, `response`).
   - **Переменная для ответа с ошибкой** — введите системное имя переменной, в которую будет помещён ответ сервера в случае ошибки (например, `errorResponse`).

     ![Настройка действия сценария для отправки HTTP-запроса](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_scenario_send_message_settings.png)

     Настройка действия сценария для отправки HTTP-запроса
4. Добавьте и настройте действие «**Повторять по количеству объектов**».

   - **Переменная** — введите имя переменной, в которую будут помещены объекты, полученные из ответа на HTTP-запрос (например, `loopiterator`). Эта переменная будет использоваться для обращения к объектам в ответе от сервера в последующем действии «**Изменить значения атрибутов**».
   - **Атрибут или выражение для поиска объектов** — введите **формулу** вида `$$response->orders`, где `response` — имя **переменной для успешного ответа**, указанное в свойствах действия «**Отправить сообщение**», `orders` — системное имя атрибута **ответа**, указанное на вкладке «[**Атрибуты сообщений**](#http_send_example_message_attributes)» в свойствах пути передачи данных.

     ![Настройка цикла в сценарии для получения данных из ответа на HTTP-запрос](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_scenario_cycle_settings.png)

     Настройка цикла в сценарии для получения данных из ответа на HTTP-запрос
5. Внутри действия «**Повторять по количеству объектов**» добавьте действие «**Создать запись**».
6. В действии «**Создать запись**» на вкладке «**Дополнительно**» установите флажок «**Сбрасывать кэш значений**».

   ![Настройка сброса кэша значений в действии «Создать запись»](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_scenario_cache.png)

   Настройка сброса кэша значений в действии «Создать запись»
7. На вкладке «**Основные**»:

   - **Целевой шаблон записи** — выберите шаблон записи `Заказы`.
   - **Ссылка на новую запись** — выберите атрибут типа «Запись» `Заказы`.
   - **Операция со значениями** — выберите операцию `Добавить`.
8. Внутри действия «**Создать запись**» добавьте действие «**Изменить значения атрибутов**».
9. В действии «**Изменить значения атрибутов**» на вкладке «**Дополнительно**» установите флажок «**Сбрасывать кэш значений**».
10. На вкладке «**Основные**»:

    - В таблице атрибутов добавьте атрибуты, значения которых следует заполнить из полученного ответа от сервера.
    - В столбце «**Значение**» для каждого атрибута введите **формулу** вида `$$loopiterator->objectName`, где `loopiterator` — имя **переменной**, заданное в действии «**Повторять по количеству объектов**», а `objectName` — имя атрибута ответа, заданное в атрибутах сообщений при настройке пути передачи данных (см. [Настройка пути передачи данных - Атрибуты сообщений](#http_send_example_message_attributes)).

      ![Настройка действия в сценарии для передачи данных из ответа на HTTP-запроса в атрибуты шаблона записи](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_scenario_set_order_attributes.png)

      Настройка действия в сценарии для передачи данных из ответа на HTTP-запроса в атрибуты шаблона записи
11. Результирующий сценарий должен выглядеть, как показано на следующей иллюстрации.

    ![Сценарий для отправки HTTP-запроса и получения данных из ответа сервера](/platform/v6.0/administration/connections_communication_routes/rest_odata_connections/img/send_http_example_scenario_result.png)

    Сценарий для отправки HTTP-запроса и получения данных из ответа сервера
12. Проверьте работу приложения: сценарий должен отправлять HTTP-запрос и записывать данные из ответа сервера в атрибуты шаблона.

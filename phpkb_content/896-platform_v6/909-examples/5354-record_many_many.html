Атрибут типа «Запись». Настройка связи многие ко многим

В Comindware Platform предусмотрена возможность создания связи многие-ко-многим. Связь многие-ко-многим — это связь, при которой множественным записям из одного шаблона записи могут соответствовать множественные записи из другого. Примером такой связи может служить проекты, в которых участвуют сотрудники: в проектах задействовано несколько сотрудников, и каждый из них может находиться в нескольких проектах.

Способ 1: атрибут с несколькими значениями

Связь создаётся указанием на уровне атрибута возможности выбора нескольких значений — флажком «Хранить несколько значений» в свойствах атрибута типа «Запись».

Такой вариант имеет значительные ограничения при использовании:

  • нет возможности указать дополнительные переменные для участника проекта, например его роль в проекте, ставку или процент загрузки;
  • нет возможности добавить фильтр для отображения значений в атрибуте.

Способ 2: промежуточный шаблон записи

Связь настраивается с помощью трёх шаблонов записи:

  • Проекты;
  • Сотрудники;

  • промежуточный шаблон записи Участники проекта / Проектная команда с атрибутами:

    • Проект — ссылка на шаблон записи Проекты;
    • Сотрудник — ссылка на шаблон записи Сотрудники.

В шаблонах записей Проекты и Сотрудники создаются атрибуты типа «Коллекция» по соответствующим ссылкам из шаблона записи Участники проекта / Проектная команда.

При таком варианте в промежуточном шаблоне можно хранить дополнительную информацию, например срок участия, роль в проекте и т. д.

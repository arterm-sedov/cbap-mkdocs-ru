# Шаблоны моделей {: #model_templates }

<div class="admonition question" markdown="block">

## Определения {: .admonition-title #definitions }

- **Шаблоны моделей** используются для хранения элементов процессной архитектуры Архитекта в виде экземпляров.
- **Шаблон модели** служит для настройки свойств шаблона, **[атрибутов][attributes]** и элементов моделей, а также для хранения экземпляров элементов моделей.
- **Элемент модели** — это экземпляр элемента процессной архитектуры (процесс, группа процессов, задача и т.&nbsp;п.), хранящийся в соответствующем шаблоне модели.
- Каждый тип элемента процессной архитектуры имеет свой шаблон модели для создания, хранения, редактирования и удаления значений.

</div>

## Просмотр списка шаблонов моделей {: #model_templates_view .pageBreakBefore }

1. На странице [«**Администрирование**»][administration] в разделе «**Архитектура**» выберите пункт «**Шаблоны**» <i class=" fal  fa-briefcase ">‌</i>.
2. Выберите вкладку «**Шаблоны моделей**».
3. Отобразится список всех шаблонов моделей.

В списке шаблонов моделей отображаются следующие столбцы:

- **Название** — наименование шаблона модели.
- **ID** — уникальный идентификатор шаблона модели.
- **Системное имя** — системное имя шаблона модели.
- **Описание** — описание назначения шаблона модели.
- **Приложения** — список приложений, в которых используется шаблон модели.
- **Тип** — тип элемента модели (например, «Процессы», «Группы процессов»).
- **Дата создания** — дата и время создания шаблона модели.
- **Создатель** — пользователь, создавший шаблон модели.
- **В архиве** — статус архивирования шаблона модели.
- **Экземпляры** — количество экземпляров элементов модели, хранящихся в шаблоне.

!!! note "Примечание"

    Шаблоны моделей также отображаются в отдельном блоке в настройках раздела навигации.

## Свойства шаблона модели {: #model_template_properties .pageBreakBefore }

Помимо **[общих свойств][template_common_properties]** шаблоны моделей содержат системные атрибуты, которые связываются с сущностями процессной архитектуры.

Каждый шаблон модели содержит системные атрибуты для установления связей между элементами моделей. Эти атрибуты автоматически создаются при создании шаблона модели и используются для:

- установления связей между элементами различных типов моделей;
- построения иерархических связей (например, между группами процессов и процессами);
- установления связей с внешними шаблонами (шаблонами записей, аккаунтов, организационных единиц, ролей).

## Смена типа элемента {: #model_template_change_type .pageBreakBefore }

При изменении типа элемента в процессной архитектуре (например, переходе с абстрактной задачи на задачу, выполняемую пользователем) экземпляр в шаблоне абстрактных задач автоматически перемещается в шаблон задач, выполняемых пользователем.

При смене типа элемента:

- системные атрибуты переносятся в новый шаблон модели;
- отличающиеся значения сохраняются в соответствующих шаблонах;
- если пользователь вернется к предыдущему типу, все значения атрибутов выбранного типа будут восстановлены.

## Изменение диаграммы модели {: #model_template_diagram_change .pageBreakBefore }

При изменении диаграммы бизнес-процесса или бизнес-способностей система автоматически записывает дату и время изменения в системный атрибут `_lastWriteDate`.

!!! note "Примечание"

    Изменение атрибутивного состава процесса или группы процесса не считается изменением диаграммы модели.

## Использование атрибутов типа «Модель» {: #model_template_model_attribute .pageBreakBefore }

Для установления связей с шаблонами моделей используется **[атрибут типа «Модель»][attribute_model]**, который позволяет:

- связывать **[шаблоны записей][record_templates]** с шаблонами моделей;
- связывать **[шаблоны аккаунтов][account_templates]** с шаблонами моделей;
- связывать **[шаблоны организационных единиц][organizational_unit_templates]** с шаблонами моделей;
- связывать **[шаблоны ролей][role_templates]** с шаблонами моделей;
- устанавливать связи между различными шаблонами моделей.

Атрибуты типа «**Модель**» поддерживают:

- использование в **[выражениях][attribute_calculated]** и триггерах;
- вычисление значений с помощью формул или выражений на языке N3;
- отображение значений в виде простого текста или гиперссылки.

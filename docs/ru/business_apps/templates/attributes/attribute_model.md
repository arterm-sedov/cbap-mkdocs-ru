# Атрибут типа «Модель» {: #attribute_model }

## Свойства атрибута {: #attribute_model_properties }

Атрибут типа «**Модель**» содержит одну или несколько ссылок на элементы моделей Архитекта, хранящиеся в **[шаблонах моделей][model_templates]**. Ссылка представляет собой уникальный идентификатор элемента модели.

Атрибут типа «**Модель**» используется для установления связей между:

- **[шаблонами записей][record_templates]** и шаблонами моделей;
- **[шаблонами аккаунтов][account_templates]** и шаблонами моделей;
- **[шаблонами организационных единиц][organizational_unit_templates]** и шаблонами моделей;
- **[шаблонами ролей][role_templates]** и шаблонами моделей;
- различными шаблонами моделей между собой.

Помимо **[общих свойств][attribute_common_properties]** для атрибута типа «**Модель**» предусмотрены перечисленные ниже свойства.

- «**Связанный шаблон модели**» (**обязательное поле**) — укажите **[шаблон модели][model_templates]**, на который будет ссылаться данный атрибут.

    --8<-- "attribute_property_attribute_interlinking.md"

- «**Формат отображения**» — выберите способ отображения значений атрибута:
    - «**Гиперссылка**» — гиперссылка, при нажатии на которую осуществляется переход:
        - если гиперссылка ведет на Группу процессов, Процесс или Встроенный подпроцесс, то происходит переход на диаграмму сущности;
        - если гиперссылка ведет на элемент диаграммы, то происходит переход на диаграмму, где данный элемент находится, при этом элемент выделяется.
    - «**Простой текст**» — текст с наименованием элемента модели.

- «**Хранить несколько значений**» — установите этот флажок, чтобы в атрибуте можно было хранить ссылки на несколько элементов модели (сформируется связь «один ко многим»).

!!! note "Примечание"

    Атрибут типа «**Модель**» поддерживает вычисление значений с помощью формул или выражений на языке N3. Это позволяет, например, определять связи между процессами на диаграмме бизнес-способностей с использованием потока данных.

## Примеры использования {: #attribute_model_example }

!!! example "Создание связи между шаблоном записи и шаблоном модели процессов"

    **Конфигурация приложения**

    - В приложении имеется шаблон записи _«Задачи»_.
    - В шаблоне модели _«Процессы»_ хранятся элементы процессной архитектуры.
    - Атрибут _«Родительский процесс»_ в шаблоне записи _«Задачи»_:
        - **тип данных**: **Модель**.
        - **Связанный шаблон модели**: _«Процессы»_.
        - **Формат отображения**: **Гиперссылка**.
        - **Хранить несколько значений**: флажок снят.

    **Результирующее поведение**

    1. В шаблоне записи _«Задачи»_ создана запись `«Подготовка документации»`.
    2. В записи `«Подготовка документации»` в атрибуте _«Родительский процесс»_ указан процесс из шаблона модели _«Процессы»_.
    3. В таблице _«Все задачи»_ в столбце _«Родительский процесс»_ будет отображаться гиперссылка для перехода к диаграмме процесса.
    4. При нажатии на гиперссылку откроется диаграмма процесса, на которую ссылается атрибут.

!!! example "Создание вычисляемого атрибута для связанных процессов"

    **Конфигурация приложения**

    - В шаблоне модели _«Процессы»_ создан атрибут _«Связанные процессы»_:
        - **тип данных**: **Модель**.
        - **Связанный шаблон модели**: _«Процессы»_.
        - **Хранить несколько значений**: флажок установлен.
        - **Вычисляемое значение**: выражение на языке N3 для определения связанных процессов по потоку данных.

    **Результирующее поведение**

    1. На диаграмме бизнес-способностей созданы два процесса с потоком данных между ними.
    2. В свойствах первого процесса в атрибуте _«Связанные процессы»_ автоматически отображается гиперссылка на второй процесс.
    3. В свойствах второго процесса в атрибуте _«Связанные процессы»_ автоматически отображается гиперссылка на первый процесс.

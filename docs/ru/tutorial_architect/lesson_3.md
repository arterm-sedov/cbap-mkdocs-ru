# Урок 3. Моделирование процессной архитектуры, интеграция, публикация {: #tutorial_architect_lesson_3 }

## Введение {: #tutorial_architect_lesson_3_intro }

В этом уроке вы научитесь строить, интегрировать и публиковать процессную архитектуру предприятия с помощью инструментов платформы. Вы освоите моделирование групп процессов, создание диаграмм бизнес-способностей и BPMN-диаграмм, а также узнаете, как интегрировать процессы с оргструктурой и внешними системами, управлять версиями моделей и публиковать их для сотрудников.

!!! warning "Бизнес-логика"

    Моделирование процессной архитектуры позволяет формализовать ключевые бизнес-процессы, повысить прозрачность и упростить интеграцию с другими системами.

**Предусловия:** пройден урок 2 «Введение, рабочий стол, роли, навигация, дашборды».

**Расчётная продолжительность:** 60 мин.

## Темы, навыки и задания урока {: #tutorial_architect_lesson_3_taxonomy }

### Темы {: #tutorial_architect_lesson_3_topics }

- Построение процессной архитектуры: группы, диаграммы бизнес-способностей
- Моделирование процессов в BPMN 2.0
- Импорт, интеграция, публикация диаграмм

### Навыки {: #tutorial_architect_lesson_3_skills }

- Создавать и управлять процессной архитектурой предприятия с примененем общепринятых нотаций (BPMN 2.0, бизнес-способности)
- Интегрировать процессы с оргструктурой и справочниками
- Использовать инструменты импорта, версионирования и публикации моделей
- Формировать управляемую, прозрачную и масштабируемую архитектуру предприятия
- Запускать согласование, аудит, анализировать историю изменений и публиковать процессы для сотрудников
- Понимать роль процессной архитектуры как драйвера цифровой трансформации и роста бизнеса

### Задания {: #tutorial_architect_lesson_3_tasks }

- Смоделировать процесс, импортировать и опубликовать BPMN-диаграмму

## Сценарий и мотивация {: #tutorial_architect_lesson_3_scenario }

!!! note "Сценарий урока"

    Представьте, что ваша компания растёт и требует прозрачной, управляемой и масштабируемой процессной архитектуры. Ваша задача — научиться моделировать процессы по мировым стандартам (BPMN 2.0), интегрировать их с оргструктурой и внешними системами, а также публиковать и анализировать изменения для цифровой трансформации бизнеса.

## Определения {: #tutorial_architect_lesson_3_definitions }

- **Процессная архитектура** — иерархия процессов, отражающая ключевые цепочки создания ценности в компании.
- **Диаграмма бизнес-способностей** — визуализация и детализация групп процессов и их взаимосвязей.
- **BPMN-диаграмма** — исполнимая модель бизнес-процесса по международному стандарту.
- **Редактор диаграмм** — инструмент для построения BPMN-диаграмм.
- **Палитра элементов** — события, задачи, развилки, потоки, объекты данных, группы.
- **Импорт** — получение моделей из внешниих систем.
- **Версионирование** — управление изменениями моделей.
- **Публикация** — утверждение и распространение моделей для пользователей.
- **Интеграция** — связь с оргструктурой, ИТ-системами, аналитикой.
- **Группа процессов** — верхний уровень архитектуры.

</div>

## Построение процессной архитектуры: группы и диаграммы бизнес-способностей {: #tutorial_architect_lesson_3_groups }

В платформе {{ productNameEnterprise }} процессная архитектура строится иерархически: группы процессов отражают ключевые направления деятельности, а диаграммы бизнес-способностей — взаимосвязи между ними.

!!! tip "Совет"
    Начинайте с выделения основных групп процессов (например, «Продажи», «HR», «Производство»), чтобы структурировать архитектуру компании.

_![Пример диаграммы бизнес-способностей](img/lesson_3_business_capabilities.png)_

1. Создайте новую группу процессов в разделе «Процессы» — «Реестр».
2. Используйте диаграмму бизнес-способностей для визуализации архитектуры.
3. Добавьте и свяжите группы, отражая цепочки создания ценности.

## Моделирование процессов в BPMN 2.0 {: #tutorial_architect_lesson_3_bpmn }

!!! note "BPMN 2.0 — международный стандарт"

    BPMN-диаграммы позволяют моделировать процессы так, чтобы их одинаково понимали бизнес, аналитики и ИТ. В {{ productNameEnterprise }} редактор диаграмм поддерживает все основные элементы BPMN.

_![Редактор BPMN-диаграмм](img/lesson_3_bpmn_editor.png)_

1. В группе процессов создайте новый процесс.
2. Откройте редактор диаграмм: добавьте начальное событие, задачи, развилки («или/или», «и»), конечное событие.
3. Используйте палитру элементов: поток управления, поток сообщений, объект данных, BPMN-группа и др.
4. Свяжите задачи с оргединицами (исполнителями), справочниками, KPI, ИТ-системами.
5. Добавьте пользовательские атрибуты, ссылки на внешние системы.

Импортируем готовую BPMN-диаграмму для ускорения моделирования и свяжем её с элементами оргструктуры.

## Версионирование, согласование и публикация моделей {: #tutorial_architect_lesson_3_versioning }

!!! tip "Совет"

    Используйте версионирование для контроля изменений и согласование для утверждения моделей перед публикацией.

1. Создайте новую версию процесса через дублирование.
2. Внесите изменения и отправьте процесс на согласование (укажите описание изменений, выберите согласующих).
3. Отслеживайте статус согласования и работайте с историей изменений.
4. После согласования опубликуйте модель для пользователей, настройте контроль доступа и уведомления.

_![Публикация модели процесса](img/lesson_3_publish_model.png)_

## Интеграция и анализ {: #tutorial_architect_lesson_3_integration }

1. Свяжите процессы с оргструктурой, справочниками, ИТ-системами.
2. Используйте дашборды для анализа процессов по статусам, KPI, аудитам.
3. Применяйте фильтры по юрлицам, подразделениям, периодам.
4. Оцените влияние изменений архитектуры на бизнес-метрики.

_![Дашборд анализа процессов](img/lesson_3_dashboard_analysis.png)_

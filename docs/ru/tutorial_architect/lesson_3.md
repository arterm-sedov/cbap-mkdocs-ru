# Урок 3. Моделирование процессной архитектуры, интеграция, публикация {: #lesson_3 }

!!! warning "Бизнес-логика"
    Моделирование процессной архитектуры позволяет формализовать ключевые бизнес-процессы, повысить прозрачность и упростить интеграцию с другими системами.

**Предусловия:** пройден урок 2 «Введение, рабочий стол, роли, навигация, дашборды».

**Расчётная продолжительность:** 60 мин.

## Темы, навыки и задания урока {: #lesson_3_taxonomy }

### Темы {: #lesson_3_topics }

- Построение процессной архитектуры: группы, диаграммы бизнес-способностей
- Моделирование процессов в BPMN 2.0
- Импорт/экспорт, интеграция, публикация

### Навыки {: #lesson_3_skills }

- Создавать и управлять процессной архитектурой
- Интегрировать процессы с оргструктурой и внешними системами

### Задания {: #lesson_3_tasks }

- Смоделировать процесс, импортировать/экспортировать BPMN-диаграмму, провести публикацию

## Сценарий и мотивация {: #lesson_3_scenario }

!!! note "Сценарий урока"
    Представьте, что ваша компания растёт и требует прозрачной, управляемой и масштабируемой процессной архитектуры. Ваша задача — научиться моделировать процессы по мировым стандартам (BPMN 2.0), интегрировать их с оргструктурой и внешними системами, а также публиковать и анализировать изменения для цифровой трансформации бизнеса.

## Определения {: #lesson_3_definitions }

- **Процессная архитектура** — иерархия процессов, отражающая ключевые цепочки создания ценности в компании.
- **Диаграмма бизнес-способностей** — визуализация групп процессов и их взаимосвязей.
- **BPMN-диаграмма** — исполнимая модель бизнес-процесса по международному стандарту.
- **Редактор диаграмм** — инструмент для построения BPMN-диаграмм.
- **Палитра элементов** — события, задачи, развилки, потоки, объекты данных, группы.
- **Импорт/экспорт** — обмен моделями и регламентами с внешними системами.
- **Версионирование** — управление изменениями моделей.
- **Публикация** — утверждение и распространение моделей для пользователей.
- **Интеграция** — связь с оргструктурой, ИТ-системами, аналитикой.

## Цели урока {: #lesson_3_goals }

- Научиться создавать и управлять процессной архитектурой предприятия по мировым стандартам (BPMN 2.0, бизнес-способности)
- Показать интеграцию процессов с оргструктурой, справочниками и внешними системами
- Освоить инструменты импорта, экспорта, версионирования и публикации моделей для цифровой трансформации бизнеса
- Получить навыки построения управляемой, прозрачной и масштабируемой архитектуры компании

## 1. Построение процессной архитектуры: группы и диаграммы бизнес-способностей {: #lesson_3_groups }

В платформе {{ productName }} процессная архитектура строится иерархически: группы процессов отражают ключевые направления деятельности, а диаграммы бизнес-способностей — взаимосвязи между ними.

!!! tip "Совет"
    Начинайте с выделения основных групп процессов (например, «Продажи», «HR», «Производство»), чтобы структурировать архитектуру компании.

_![Пример диаграммы бизнес-способностей](img/lesson_3_business_capabilities.png)_

**Практика:**
- Создайте новую группу процессов в разделе «Процессы» — «Реестр».
- Используйте диаграмму бизнес-способностей для визуализации архитектуры.
- Добавьте и свяжите группы, отражая цепочки создания ценности.

## 2. Моделирование процессов в BPMN 2.0 {: #lesson_3_bpmn }

!!! note "BPMN 2.0 — международный стандарт"
    BPMN-диаграммы позволяют моделировать процессы так, чтобы их одинаково понимали бизнес, аналитики и ИТ. В {{ productName }} редактор диаграмм поддерживает все основные элементы BPMN.

_![Редактор BPMN-диаграмм](img/lesson_3_bpmn_editor.png)_

**Пошагово:**
1. В группе процессов создайте новый процесс.
2. Откройте редактор диаграмм: добавьте начальное событие, задачи, развилки («или/или», «и»), конечное событие.
3. Используйте палитру элементов: поток управления, поток сообщений, объект данных, BPMN-группа и др.
4. Свяжите задачи с оргединицами (исполнителями), справочниками, KPI, ИТ-системами.
5. Добавьте пользовательские атрибуты, ссылки на внешние системы.

!!! question "Рефлексия"
    Как использование BPMN-диаграмм помогает вам структурировать и оптимизировать процессы в вашей компании?

## 3. Импорт и экспорт BPMN-диаграмм {: #lesson_3_import_export }

!!! note "Практика"
    Импортируйте готовую BPMN-диаграмму (например, «Поиск и найм кандидатов.bpmn») для ускорения моделирования. Экспортируйте модели для обмена с внешними системами или формирования регламентов.

**Пошагово:**
- В реестре процессов выберите «Импорт» и загрузите BPMN-диаграмму.
- Проверьте структуру, настройте связи с оргструктурой.
- Для обмена используйте экспорт в BPMN, PNG, SVG.
- Экспортируйте регламент процесса с помощью шаблона (DOCX, PDF).

_![Импорт BPMN-диаграммы](img/lesson_3_import_bpmn.png)_

## 4. Версионирование, согласование и публикация моделей {: #lesson_3_versioning }

!!! tip "Совет"
    Используйте версионирование для контроля изменений и согласование для утверждения моделей перед публикацией.

**Пошагово:**
- Создайте новую версию процесса через дублирование.
- Внесите изменения, отправьте на согласование (описание изменений, выбор согласующих).
- Отслеживайте статус, работайте с историей изменений.
- После согласования опубликуйте модель для пользователей, настройте контроль доступа и уведомления.

_![Публикация модели процесса](img/lesson_3_publish_model.png)_

## 5. Интеграция и анализ {: #lesson_3_integration }

!!! note "Практика"
    Интегрируйте процессы с оргструктурой, справочниками, ИТ-системами для сквозной автоматизации. Используйте дашборды для анализа процессов по статусам, KPI, аудитам.

**Пошагово:**
- Свяжите процессы с оргструктурой, справочниками, ИТ-системами.
- Используйте дашборды для анализа процессов по статусам, KPI, аудитам.
- Применяйте фильтры по юрлицам, подразделениям, периодам.
- Оцените влияние изменений архитектуры на бизнес-метрики.

_![Дашборд анализа процессов](img/lesson_3_dashboard_analysis.png)_

## Ключевые понятия {: #lesson_3_concepts }

- **Группа процессов** — верхний уровень архитектуры
- **Диаграмма бизнес-способностей** — детализация групп процессов
- **BPMN-диаграмма** — исполнимая модель процесса
- **Редактор диаграмм** — инструмент для построения BPMN
- **Палитра элементов** — события, задачи, развилки, потоки, объекты данных, группы
- **Импорт/экспорт** — обмен моделями и регламентами
- **Версионирование** — управление изменениями моделей
- **Публикация** — утверждение и распространение моделей
- **Интеграция** — связь с оргструктурой, ИТ-системами, аналитикой

## Ожидаемые результаты {: #lesson_3_outcomes }

- Вы умеете создавать и управлять процессной архитектурой, соответствующей лучшим мировым практикам
- Интегрируете процессы с оргструктурой, справочниками и ИТ-системами
- Осваиваете инструменты импорта, экспорта, версионирования и публикации моделей
- Запускаете согласование, аудит, анализируете историю изменений и публикуете процессы для сотрудников
- Понимаете роль процессной архитектуры как драйвера цифровой трансформации и роста бизнеса
- Готовы внедрять изменения, которые делают компанию гибкой, прозрачной и конкурентоспособной

    Или импортируйте BPMN-диаграмму из внешней системы, настройте интеграцию с оргструктурой и проанализируйте, как изменения архитектуры влияют на бизнес-метрики в аналитике платформы.

# Урок 3. Моделирование процессной архитектуры, интеграция, публикация {: #tutorial_architect_lesson_3 }

## Введение {: #tutorial_architect_lesson_3_intro }

В этом уроке вы научитесь строить, интегрировать и публиковать процессную архитектуру предприятия с помощью инструментов платформы. Вы освоите моделирование групп процессов, создание диаграмм бизнес-способностей и BPMN-диаграмм, а также узнаете, как интегрировать процессы с оргструктурой и внешними системами, управлять версиями моделей и публиковать их для сотрудников.

!!! warning "Бизнес-логика"

    Моделирование процессной архитектуры позволяет формализовать ключевые бизнес-процессы, повысить прозрачность и упростить интеграцию с другими системами.

**Предусловия:** пройден _«[Урок 2. Рабочий стол, навигация, роли, оргструктура][tutorial_architect_lesson_2]»_.

**Расчётная продолжительность:** 60 мин.

## Темы, навыки и задания урока {: #tutorial_architect_lesson_3_taxonomy }

### Темы {: #tutorial_architect_lesson_3_topics }

- Построение процессной архитектуры: группы, диаграммы бизнес-способностей
- Моделирование процессов в BPMN 2.0
- Импорт, интеграция, публикация диаграмм

### Навыки {: #tutorial_architect_lesson_3_skills }

- Интегрировать процессы с оргструктурой и справочниками
- Использовать инструменты импорта, версионирования и публикации моделей
- Формировать управляемую, прозрачную и масштабируемую архитектуру предприятия
- Запускать согласование, аудит, анализировать историю изменений и публиковать процессы для сотрудников

## Определения {: #tutorial_architect_lesson_3_definitions }

- **Процессная архитектура** — иерархия процессов, отражающая ключевые цепочки создания ценности в компании.
- **Диаграмма бизнес-способностей** — визуализация и детализация групп процессов и их взаимосвязей.
- **BPMN-диаграмма** — исполнимая модель бизнес-процесса по международному стандарту.
- **Редактор диаграмм** — инструмент для построения BPMN-диаграмм.
- **Палитра элементов** — события, задачи, развилки, потоки, объекты данных, группы.
- **Импорт** — получение моделей из внешниих систем.
- **Версионирование** — управление изменениями моделей.
- **Публикация** — утверждение и распространение моделей для пользователей.
- **Интеграция** — связь с оргструктурой, ИТ-системами, аналитикой.
- **Группа процессов** — верхний уровень архитектуры.

</div>

## Построение процессной архитектуры: группы и диаграммы бизнес-способностей {: #tutorial_architect_lesson_3_groups }

В **{{ productNameEnterprise }}** процессная архитектура строится иерархически: группы процессов отражают ключевые направления деятельности, а диаграммы бизнес-способностей — взаимосвязи между ними.

!!! tip "Совет"

    Начинайте с выделения основных групп процессов (например, «Продажи», «HR», «Производство»), чтобы структурировать архитектуру компании.

<!-- _![Пример диаграммы бизнес-способностей](img/tutorial_architect_lesson_3_business_capabilities.png)_ -->

1. Перейдите в раздел «**Процессы**» в навигации.
2. Нажмите кнопку «**Создать**» <i class="fa-light fa-plus"></i>.
3. Нажмите кнопку «**Группа процессов**» <i class=" fal  fa-list-tree " aria-hidden="true"></i>.
4. Откроется **конструктор** группы процессов.
5. Нажмите кнопку «**Свойства**» <i class="fa-light fa-sidebar-flip"></i>.
6. Введите **наименование** группы процессов _«Поиск и найм персонала»_.
7. Нажмите кнопку «**Редактировать**» <i class="fa-light fa-pen-nib"></i>.
8. Добавьте на диаграмму новый **процесс**.
9. Переименуйте процесс в _«Поиск кандидата»_.
10. Добавьте на диаграмму новый процесс и переименуйте его в _«Найм кандидата»_.
11. Добавьте на диаграмму новый **ресурс**.
12. Переименуйте ресурс в _«Кандидаты»_.
13. Нажмите на элемент _«Поиск кандидата»_.
14. В меню быстрого создания нажмите «**Поток данных**».
15. Соедините поток данных с ресурсом _«Кандидаты»_.
16. Соедините потоком данных ресурс _«Кандидаты»_ с процессом _«Найм кандидата»_.

## Моделирование процесса «Поиск кандидата» {: #tutorial_architect_lesson_3_bpmn }

1. Откройте реестр процессов.
2. Дважды нажмите строку процесса _«Поиск кандидата»_.
3. Откроется конструктор процесса.
4. Нажмите кнопку «**Редактировать**» <i class="fa-light fa-pen-nib"></i>.
5. Добавьте на диаграмму «**Основной пул**».
6. Переименуйте основной пул в _«Поиск кандидата»_.
7. Добавьте дорожку и переименуйте её в _«Рекрутер»_.
8. Добавьте начальное событие и переименуйте его в _«Поступила заявка на поиск кандидата»_.
9. Нажмите событие _«Поступила заявка на поиск кандидата»_.
10. Измените тип события на «**Событие-получение сообщения**».
11. Добавьте в пул **абстрактную задачу** и переименуйте её в _«Создать вакансию»_.
12. Измените тип задачи _«Создать вакансию»_ на **пользовательскую**.
13. Нажмите событие _«Поступила заявка на поиск кандидата»_.
14. В меню быстрого создания нажмите «**Поток управления**».
15. Соедините поток управления с задачей _«Создать вакансию»_.
16. Добавьте в пул **объект данных** и переименуйте его в _«Вакансия»_.
17. Соедините объект _«Вакансия»_ с задачей _«Создать вакансию»_ потоком данных.
18. Добавьте **развилку «или/или»**.
19. Соедините задачу _«Создать вакансию»_ с **развилкой «или/или»**.
20. Добавьте пользовательскую задачу _«Искать кандидата в базах данных»_.
21. Соедините **развилку «или/или»** с задачей _«Искать кандидата в базах данных»_.
22. добавьте промежуточное событие _«Поступило резюме»_.
23. Измените тип события _«Поступило резюме»_ на «**Событие-условие**».
24. Соедините задачу _«Искать кандидата в базах данных»_ с событием _«Поступило резюме»_.
25. После события _«Поступило резюме»_ добавьте пользовательскую задачу _«Договориться с кандидатом о собеседовании»_.
26. После задачи _«Договориться с кандидатом о собеседовании»_ добавьте **развилку «или/или»** и переименуйте её в _«Кандидат согласен пройти собеседование?»_.
27. От развилки _«Кандидат согласен пройти собеседование?»_ добавьте **поток управления** _«Нет»_ и соедините его с **развилкой «или/или»**.
28. От развилки _«Кандидат согласен пройти собеседование?»_ добавьте **поток управления** _«Да»_ и соедините его с простым **конечным событием**.
29. Добавьте **скрытый пул**.
30. Выберите связываемый процесс _«Найм кандидата»_.
31. Соедините потоком сообщений конечное событие со скрытым пулом _«Найм кандидата»_.

## Импорт процесса «Найм кандидата» {: #tutorial_architect_lesson_3_import_export }

Импортируем готовую BPMN-диаграмму для ускорения моделирования и свяжем её с элементами оргструктуры.

1. ЗАгрузите на свой компьютер файл `Найм кандидата.bpmn`.
2. Откройте реестр процессов.
3. Дважды нажмите строку процесса _«Найм кандидата»_.
4. Откроется конструктор процесса.
5. Нажмите кнопку «**Редактировать**» <i class="fa-light fa-pen-nib"></i>.
6. Нажмите кнопку «**Импорт**» <i class="fa-light fa-file-import"></i>.
7. Выберите файл `Найм кандидата.bpmn`.
8. Нажмите кнопку «**Продолжить**».
9. Проверьте, что на процессе _«Найм кандидата»_ установлен флажок.
10. Нажмите кнопку «**Импортировать**».
11. Диаграмма процесса _«Найм кандидата»_ будет импортирована.
12. Проверьте и при необходимости отредактируйте диаграмму.

<!-- _![Импорт BPMN-диаграммы](img/tutorial_architect_lesson_3_import_bpmn.png)_ -->

!!! example "Факультативное задание"

    Добавьте в группу процессов _«Поиск и найм кандидата»_ другие процессы, например _«Внесение заявки на открытие вакансии»_.

## Добавление регламента процессов {: #tutorial_architect_lesson_3_versioning }

1. Откройте процесс _«Поиск кандидата»_ для редактирования.
2. Откройте **свойства** процесса.
3. Добавьте владельца процесса, например свой аккаунт.
4. Укажите _Рекрутера_ как **участника процесса**.
5. Нажмите кнопку «**Экспорт**» <i class="fa-light fa-file-export"></i>.
6. Нажмите кнопку «**Экспорт регламента**».
7. Скачайте регламент процесса в формате Word.
8. В **свойствах** процесса добавьте скачанный файл в поле «**Регламент в Word**».

!!! example "Факультативное задание"

    Настройте процесс _«Найм кандидата»_ аналогичным образом.

# Урок 2. Введение, рабочий стол, роли, навигация, дашборды {: #lesson_2 }

!!! warning "Бизнес-логика"
    Эффективная навигация и понимание ролей в системе позволяют быстро адаптироваться к изменениям и минимизировать ошибки при работе с корпоративной архитектурой.

**Предусловия:** пройден урок 1 «Обзор возможностей платформы и понятий BPMS».

**Ожидаемое время прохождения:** 45 мин.

## Темы, навыки и задания урока {: #lesson_2_taxonomy }
### Темы
- Роли пользователей: Архитектор и Сотрудник
- Интерфейс: рабочий стол, навигация, быстрые действия, дашборды
### Навыки
- Ориентироваться в интерфейсе платформы
- Использовать быстрые действия и аналитику
### Задания
- Войти под разными ролями, изучить интерфейс, выполнить практические задания

## Сценарий и мотивация {: #lesson_2_scenario }

!!! note "Сценарий урока"
    Представьте, что вы — новый архитектор или сотрудник, которому нужно быстро освоить работу с корпоративной архитектурой компании. Ваша задача — научиться ориентироваться в интерфейсе, использовать рабочий стол, быстрые действия и аналитику для эффективного старта.

## Определения {: #lesson_2_definitions }

- **Архитектор** — пользователь, моделирующий процессы, настраивающий оргструктуру, согласующий и публикующий изменения.
- **Сотрудник** — пользователь, просматривающий оргструктуру, участвующий в согласованиях и аудитах, работающий с задачами и документами.
- **Рабочий стол** — стартовая страница с быстрыми действиями и аналитикой.
- **Быстрые действия** — плитки для быстрого запуска ключевых операций.
- **Дашборды** — визуализация процессов, заявок, аудитов, динамики изменений.
- **Навигация** — переход между основными разделами платформы.

## Цели урока {: #lesson_2_goals }
- Понять назначение и возможности модуля
- Освоить различия между ролями Архитектор и Сотрудник
- Научиться работать с рабочим столом, навигацией, быстрыми действиями и дашбордами
- Получить стартовые навыки для самостоятельной работы с архитектурой предприятия

## 2. Знакомство с рабочим столом {: #lesson_2_desktop }

!!! note "Навигация по рабочему столу"
    Виджет «С чего начать» поможет новичкам быстро приступить к работе. Быстрые действия позволяют мгновенно создавать согласования, аудиты и другие операции.

_![Рабочий стол с быстрыми действиями](img/lesson_2_dashboard.png)_

**Пример:**
- Нажмите на плитку «Согласовать процесс» — откроется форма для создания нового согласования.
- Используйте «Назначить аудит» для запуска проверки процесса или подразделения.

## 3. Навигация по разделам платформы {: #lesson_2_navigation }

!!! tip "Изучайте интерфейс"
    Переключайтесь между разделами, чтобы изучить их структуру и назначение. Обратите внимание на различия в интерфейсе для разных ролей.

_![Панель навигации](img/lesson_2_nav_panel.png)_

Навигационная панель слева позволяет быстро переходить между основными разделами:
- **Архитектор:** Рабочий стол, Управление архитектурой, Оргструктура, Процессы, Управление версиями
- **Сотрудник:** Рабочий стол, Телефонный справочник, Мой профиль, Моя оргструктура, Мои процессы

## 4. Быстрые действия и виджеты {: #lesson_2_quick_actions }

!!! note "Практика"
    Откройте форму создания согласования через быстрые действия. Кликните по диаграмме в отчёте, чтобы перейти к деталям.

_![Плитки быстрых действий](img/lesson_2_quick_actions.png)_

Быстрые действия — это плитки на рабочем столе, которые позволяют быстро запускать ключевые операции. Например, создание согласования или аудита занимает всего пару кликов. Виджеты отображают актуальную аналитику: количество процессов, заявок, аудитов, динамику изменений.

**Практика:**
- Откройте форму создания согласования через быстрые действия.
- Кликните по диаграмме в отчёте, чтобы перейти к деталям.

## 5. Дашборды и аналитика {: #lesson_2_dashboards }

!!! example "Рефлексия"

    Как дашборды помогают вам принимать решения и контролировать процессы?

_![Пример дашборда](img/lesson_2_dashboard_example.png)_

Дашборды и отчёты — мощный инструмент для анализа процессов и оргструктуры. Вы можете видеть динамику заявок, статусы аудитов, использовать фильтры по периоду, юрлицу или ИТ-системе. Клик по диаграмме позволяет перейти к подробной информации.

## Ключевые понятия {: #lesson_2_concepts }
- **Архитектор** — создание, моделирование, согласование, аудит, публикация
- **Сотрудник** — просмотр, участие в согласованиях и аудитах, работа с задачами
- **Рабочий стол** — стартовая страница с быстрыми действиями и аналитикой
- **Быстрые действия** — плитки для создания согласований, аудитов и других операций
- **Дашборды и отчёты** — визуализация процессов, заявок, аудитов, динамики изменений
- **Навигация** — переход между основными разделами платформы

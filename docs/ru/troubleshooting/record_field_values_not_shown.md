# Не отображаются значения в ссылочном поле {: #record_field_values_not_shown }

## Описание проблемы {: #record_field_values_not_shown_intro }

В шаблоне записи предусмотрен ссылочный атрибут типа «**Запись**» для выбора значений из списка записей другого шаблона записи.

Иногда у пользователя может возникнуть ситуация, когда нет возможности выбора на форме ни одного из значений.

Одной из причин может быть некорректно настроенное свойство отображаемого атрибута в **связанном шаблоне** записи.

Другой причиной может быть неполная настройка разрешений в **ролях**.

## Решение {: #solution }

1. Для проверки перейдите к свойствам **связанного шаблона** записи, на который ссылается атрибут
2. Откройте список атрибутов шаблона.
3. Найдите атрибут, значения которого должны отображаться показываться в ссылках на данный шаблон, — атрибут, используемый как **заголовок записей**. Как правило, это текстовое поле с названием записи, которое однозначно идентифицирует её, например, название контрагента проекта или договора.
4. Если атрибут-заголовок записи является вычисляемым, проверьте правильность выражения.
5. Если п. 3–4 проработаны, а значения в ссылочном поле не отображаются, проверьте настройку разрешений в роли пользователя. На **связанный шаблон**, на который ссылается атрибут типа «**Запись**», должны быть выданы права на просмотр.

!!! note "Примечание"

    При отсутствии атрибута, **используемого как заголовок записей**, по умолчанию в ссылочном поле будут отображаться ID записей.

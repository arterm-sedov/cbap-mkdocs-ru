
Для контроля состояния кластера и безопасного выполнения перезапуска могут потребоваться установленные скрипты Apache Ignite
Установка скриптов производится согласно инструкции в директорию `/usr/share/ignite`

### Остановка узлов кластера
#### На останавливаемом узле:
- выполнить в терминале команду 
```sh
systemctl stop comindware<instanceName>
```

#### На остающемся в кластере узле:
- выполнить команду
```sh
bash /usr/share/ignite/bin/control.sh
```
 дождаться выхода узла из топологии
 
#### На останавливаемом узле:
- дождаться полной остановки экземпляра приложения

```sh
systemctl status comindware<instanceName>
```

#### При необходимости произвести вышеперечисленные действия для остальных узлов кластера

### Запуск узлов кластера

#### На запускаемом узле:
- выполнить в терминале команду 
```sh
systemctl start comindware<instanceName>
```

#### На первом поднятом узле кластера:
- выполнить команду
```sh
bash /usr/share/ignite/bin/control.sh
```
Дождаться входа узла в топологию
 
#### В веб-Браузере:
- выполнить запрос на узел Приложения, дождаться отображения веб-интерфейса, пройти авторизацию в Платформе
#### При необходимости произвести вышеперечисленные действия для остальных узлов кластера

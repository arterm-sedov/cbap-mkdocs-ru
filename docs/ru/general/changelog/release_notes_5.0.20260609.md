# Сведения о выпуске 5.0 Q1 2026 от 09.06.2026 {: #release_notes_5.0.20260609 }

## Введение {: #release_notes_5.0.20260609_intro }

**{{ productName }}** версии 5.0 Q1 2026 от 09.06.2026 включает ряд улучшений и исправлений по сравнению с [версией 5.0 от 31.12.2025][release_notes_5.0.20251231].

В этом выпуске мы сосредоточились на исправлении проблем, выявленных в клиентских решениях, а также на развитии поиска, кластера, интеграций, администрирования, лицензирования и средств диагностики. Значительная часть изменений направлена на более предсказуемую работу с большими объёмами данных, повышение стабильности интеграционных сценариев и расширение журналирования.

!!! warning "Обновление до версии 5.0 Q1 2026"

    Из-за изменения структуры конфигурационных файлов обновление до версии 5.0 Q1 2026 следует выполнять только посредством создания нового экземпляра ПО и подключения к нему базы данных обновляемого экземпляра.

## Ключевые изменения {: #release_notes_5.0.20260609_key_changes }

- **Клиентские решения:** исправлены проблемы, выявленные при эксплуатации клиентских решений и обработанные в приоритетном порядке. <!-- ASAP -->
- **Интеграции:** добавлены адаптеры для импорта и экспорта данных через {{ apacheKafkaVariants }}, доработана архитектура адаптеров, улучшены SQL-коннектор и импорт данных из Excel. <!-- 10560845, 10366590, 10557913, 10498881 -->
- **Кластер:** улучшена работа API Gateway в кластерной конфигурации. <!-- 10405663 -->
- **Поиск, таблицы и данные:** улучшены фильтрация, детализация данных на виджетах, перенос текста в таблицах, фиксация списков на формах и работа древовидных таблиц коллекций. <!-- 10245491, 10394348, 1880799, 10516611 -->
- **Администрирование и аудит:** расширено журналирование действий над аккаунтами и группами в `IDM.log`, добавлена возможность пользователям самостоятельно назначать замещающего сотрудника. <!-- 10613670, 10340067 -->
- **Лицензирование:** доработаны механизмы лицензирования платформы. <!-- 1932839 -->

## Новые возможности и улучшения {: #release_notes_5.0.20260609_new_and_improved }

### Интеграции, API и обмен данными {: #release_notes_5.0.20260609_integrations_api }

- **Адаптеры {{ apacheKafkaVariants }}:** добавлены адаптеры импорта данных в платформу и экспорта данных из платформы через {{ apacheKafkaVariants }}. Адаптеры настраиваются в подключениях к шине сообщений и могут работать без сценария получения сообщения. <!-- 10560845 -->
- **Архитектура адаптеров:** изменена архитектура работы с адаптерами для повышения гибкости и устойчивости интеграционных сценариев. Встроенные и внешние адаптеры унифицированы через сервис `adapterhost`, а {{ apacheKafkaVariants }} используется для хранения данных адаптеров, подключений, путей передачи данных и их состояния. <!-- 10366590 -->
- **SQL-коннектор:** добавлена поддержка различных условий сравнения данных в фильтрах SQL-коннектора. <!-- 10557913 -->
- **Импорт из Excel:** добавлена возможность указать произвольный атрибут в качестве ключа при импорте данных из Excel в поля типа «Запись». <!-- 10498881 -->
- **API Gateway:** повышена стабильность работы API Gateway в кластере. <!-- 10405663 -->

### Интерфейс, таблицы и поиск {: #release_notes_5.0.20260609_ui_search }

- **Таблицы:** реализован перенос текста в ячейках таблиц для более удобного просмотра длинных значений. <!-- 10394348 -->
- **Древовидные таблицы коллекций:** исправлено поведение настройки древовидных таблиц в коллекциях. <!-- 10516611 -->
- **Формы:** при переходе из большого списка к форме записи и обратно список сохраняет прежнюю позицию прокрутки. <!-- 1880799 -->
- **Виджеты:** улучшена фильтрация данных в детализации данных на виджетах страниц. <!-- 10245491 -->
- **Денежный формат:** добавлено отображение счётчиков и диаграмм на рабочем столе в денежном формате. <!-- 10448948 -->

### Администрирование, аудит и лицензирование {: #release_notes_5.0.20260609_admin_audit }

- **Журнал `IDM.log`:** добавлено логирование всех действий над аккаунтами и группами. <!-- 10613670 -->
- **Замещения сотрудников:** добавлена персональная возможность назначать замещающего сотрудника. Возможность включается параметром `account.selfSubstitutionsEnabled` в конфигурационном файле. <!-- 10340067 -->
- **Лицензирование:** доработаны механизмы лицензирования в платформе. <!-- 1932839 -->

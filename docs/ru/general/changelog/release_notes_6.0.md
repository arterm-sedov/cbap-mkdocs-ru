## Введение {: #release_notes_6.0_intro }

**{{ productName }}** версии 6.0 включает ряд новых возможностей, улучшений и исправлений по сравнению с [версией 5.0 от 31.12.2025][release_notes_5.0.20251231].

В этом выпуске мы сосредоточились на:

- развитии интерфейса;
- мобильном приложении;
- **{{ productNameArchitect }}**;
- интеграциях;
- администрировании;
- производительности и устойчивости инфраструктуры.

Значительная часть изменений направлена на:
- более предсказуемую работу с большими объёмами данных;
- стабильное выполнение процессов;
- улучшение пользовательских сценариев на формах, страницах и в мобильном приложении;
- развитие средств настройки, журналирования и диагностики.

## Новые возможности и улучшения {: #release_notes_6.0_new_and_improved }

### ИИ {: #release_notes_6.0_ui }

#### 1. Входная точка

- Слушает Kafka-топик с запросами пользователей.
- Извлекает текст запроса, контекст, `user_id`, `solution_id` из входящих сообщений.
- Передаёт данные в основной обработчик.

#### 2. Классификация намерений

Агент определяет тип действия через LLM:

- **«Навигация»** — открытие таблиц, карточек, поиск, просмотр данных.
- **«Заполнение формы»** — создание / редактирование записей.
- **«Неизвестно»** — общие вопросы, на которые отвечает LLM напрямую.

#### 3. Обработка навигации

При типе «Навигация» агент:

- Определяет подтип: таблица, запись, задача, виджет, дашборд, глобальный поиск.
- Проверяет контекст: если пользователь находится на открытой странице (таблица / дашборд), обрабатывает контекстные действия:
  - фильтрация открытой таблицы (например, *«отфильтруй по клиенту Альфа»*);
  - сортировка (*«отсортируй по дате»*);
  - группировка.
- Формирует навигационные операции:
  - открытие таблиц с фильтрами, сортировкой, группировкой;
  - открытие карточек записей;
  - глобальный поиск;
  - открытие дашбордов с диаграммами.

#### 4. Заполнение форм

При типе «Заполнение формы»:

- Извлекает данные из запроса пользователя.
- Определяет поля формы и их значения.
- Формирует операцию заполнения с `widgetChanges`.
- Поддерживает создание новых записей и редактирование существующих.

#### 5. Выходные данные

Агент возвращает структуру:

```json
{
    "success": bool,
    "response_text": str,
    "operations": dict,
    "intent": str,
    "metadata": dict
}
```

#### 6. Отправка результата

- Формирует ответ в формате `Comindware (NavigateOperation)`.
- Отправляет в Kafka-топик для ответов.
- Логирует все отправленные сообщения в файлы.

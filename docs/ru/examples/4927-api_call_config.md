# Настройка вызова API {: #api_call_config }

В **{{ productName }}** есть возможность вызова из внешних систем веб-сервисов системы для обмена данными, например, создание объекта, запуск процесса или получение информации по документу. Полный перечень веб-сервисов и описание можно посмотреть [здесь][api_intro].

## Сценарий {: #api_call_config_scenariy }

Нужно запустить процесс по заявке на командировку с информацией: страна, город и аэропорт.

## Исходные данные {: #api_call_config_opisanie }

В системе созданы:

- шаблон записи _«Заявки на командировки»_ с атрибутами:

    - _Страна_ (`op.1`);
    - _Город_ (`op.2`);
    - _Аэропорт_ (`op.3`).

- шаблон процесса _«Обработка заявки на командировки»_ (`pa.1`), связанный с шаблоном записи _«Заявки на командировки»_.

## Порядок настройки {: #api_call_config_setup }

Для запуска процесса используйте веб-сервис:

- `https://URL/api/public/system/Process/ProcessObjectService/Create1`
- **Метод аутентификации** — `Basic`. Большинство внешних систем обладают интерфейсом настройки метода аутентификации. При отсутствии такового добавьте в заголовок `Header` запроса параметр `Authorization`, равный зашифрованному в формате Base64 логину и паролю.
- **Тело запроса** — укажите идентификатор процесса, который нужно запустить, а также идентификаторы и значения атрибутов, которые вы хотите передать:

```json
{
  "processAppId": "pa.1",
  "objectName": null,
  "syncActivityQuantity": 2,
  "objectData": {
    "op.1": "Россия",
    "op.2": "Москва",
    "op.3": "Шереметьево"
  }
}
```

!!! note "Примечание"

    У пользователя, под которым выполняется запрос, должны быть соответствующие права.

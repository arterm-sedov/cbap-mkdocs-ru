# Вычисление аккаунтов, входящих в роль {: #example_n3_calculate_role_accounts }

## Прикладная задача {: #example_n3_calculate_role_accounts_use_case }

Требуется получить список пользователей, входящих в определённую роль, например для автоматического назначения задачи.

Результат вычисления необходимо поместить в атрибут типа «**Аккаунт**», в свойствах которого установлен флажок «**Хранить несколько значений**».

## Настройка вычисления {: #example_n3_calculate_role_accounts_configure }

1. Создайте или откройте атрибут типа «**Аккаунт**», в который требуется поместить список пользователей, входящих в роль.
2. В свойствах атрибута установите флажки:
    - «**Хранить несколько значений**»
    - «**Вычислять автоматически**»
3. Введите следующее **вычисляемое значение** на языке **N3**:

    ``` turtle
    @prefix cmwui: <http://comindware.com/ontology/ui#>.
    @prefix account: <http://comindware.com/ontology/account#>.
    @prefix role: <http://comindware.com/ontology/role#>.
    {
        # Получаем роль по её ID.
        # Число после role. зависит от конкретного экземпляра {{ productName }}
        # и отображается в адресной строке браузера
        # при просмотре свойств роли.
        "role.1" cmwui:toQName ?role.
        # Получаем аккаунты и группы, входящие в роль.
        ?role role:roleMembers ?value.
        # Возвращаем только пользователей в вычисляемый атрибут.
        # Чтобы вернуть группы, замените account:Account
        # на account:Group.
        ?value a account:Account.
    }
    ```

## Пояснения {: #example_n3_calculate_role_accounts_details }

С помощью данной тройки можно получить как аккаунты, так и группы:

- `account:Account` — пользователи;
- `account:Group` — группы.

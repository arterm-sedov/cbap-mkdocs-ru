!!! warning "Резервное копирование крупных баз данных"

    Если размер базы данных превышает 10&nbsp;ГБ, рекомендуется использовать скрипт резервного копирования на сервере для обеспечения оптимальной производительности резервного копирования и работы **{{ productName }}**.
    
    Это связано с тем, что резервное копирование больших баз данных встроенными средствами **{{ productName }}** создаёт значительную вычислительную нагрузку и может приводить к снижению производительности.
    
    См. _«[Настройка и использование скрипта для резервного копирования данных (Linux)][backup_linux_script]»_.
